# The Problem

Most robot learning benchmarks live in simulation. The few real-robot evaluations that exist typically test a handful of tasks, on a single hardware setup, under conditions the model was trained on. This makes it nearly impossible to answer a basic question: what can a generalist manipulation policy actually do in the physical world?

ManipArena is designed to answer that question. It is a server-side evaluation platform where participants submit a single HTTP inference endpoint, and we run it across 20 reasoning-oriented tasks on real robots — with systematic out-of-distribution testing, partial-credit scoring, and multi-dimensional capability diagnostics.

# Controlled Environment + Systematic OOD

All evaluations run inside a green-screen enclosed booth with fixed studio lighting. This isn't aesthetic — it's methodological. By controlling the physical environment, we isolate the variables that actually matter: object properties, spatial configurations, and semantic compositions.

Each task follows a 10-trial stratified evaluation protocol. Trials T1–T4 are in-distribution. Trials T5–T8 introduce visual shifts (new positions, orientations). Trials T9–T10 test true OOD — objects the model has never seen during training. This layered design lets us chart a model's generalization curve, not just a single pass/fail number.

# Key Findings

1. Five Capability Dimensions

We identify five orthogonal capability dimensions from the results: Coarse Manipulation, Precision Manipulation, Sequential Reasoning, Semantic Understanding, and Spatial Robustness. Each model dominates a different subset — no model excels at everything.

2. VLA vs WAM: Complementary Strengths

VLA models excel at precision manipulation (put_glasses: 87/100) and semantic understanding (58.9 avg). DreamZero excels at coarse manipulation (pick_items: 97.8) and spatial robustness (-8% vs -57% on position shifts). Neither paradigm subsumes the other.

3. The Semantic Gap Depends on Instruction Completeness

On semantic tasks where the language instruction fully specifies the objective ("put headphones into the basket"), DreamZero remains competitive (score: 66). But when the model must extract the action plan from visual context (read a color-sequence card, infer matching rules), DreamZero drops to 3–13.5 while VLA maintains 13–51.5. The gap is not about "semantic reasoning" in general — it's about whether the task goal is fully specified by language alone.

4. Force-Sensitive Manipulation Is the Open Frontier

Tasks requiring force modulation — pouring, insertion, zipping — score below 30/100 for all models. The 56D action space includes motor currents specifically to enable research on this frontier. Current models barely use this signal.

5. Multi-Task Unification Helps — With Trade-offs

OneModel outperforms Single overall (640.5 vs 626.3), with a +109% gain on semantic tasks. But it pays a -73% penalty on procedural tasks like press_button. Multi-task training amplifies cross-task transfer for semantic understanding but can interfere with task-specific procedural skills.

# Beyond the Benchmark: What Happens Under Full OOD?

The main benchmark tests OOD within the training distribution's neighborhood. But what happens when every object is completely novel?

We ran a supplementary 14-trial controlled experiment using 7 unseen fruits and 3 unseen plates — none appearing in any training trajectory. The results were striking:

The VLA–WAM gap inverts under full OOD. In-distribution, VLA outperforms DreamZero on semantic tasks (58.9 vs 32.7). Under full OOD, DreamZero outperforms VLA ~8.5:1. The VLA model achieves exactly one grasp across 14 trials.

Even more surprising: DreamZero performs better with reasoning-style instructions ("the largest fruit", "the rightmost plate") than with explicit novel object names ("pineapple", "loquat"). In 3 of 4 controlled same-scene comparisons, descriptive cues outperform explicit names — suggesting that when objects are unfamiliar, relational descriptions are more actionable than novel identifiers.

This points to a fundamental architectural difference: VLA's VLM backbone relies on pattern matching against training data distributions, while DreamZero's video-diffusion backbone encodes physical priors that generalize more robustly to novel visual inputs.
